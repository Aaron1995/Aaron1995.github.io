基于Gradle的Maven项目添加第三方SDK并导入

  • Gradle
  • Spring Boot

Spring Boot项目上添加支付宝SDK

基于Gradle构建的Boot项目没有pom的xml配置信息，抛开手动的Ctrl+C Ctrl+V的菜鸟手法，就要明白如何让第三方SDK添加到本地的mvn仓库，在idea的Terminal下执行

mvn install:install-file -DgroupId=?(eg:alipay) -DartifactId=?(eg:alipay) -Dversion=?(eg:1.0) -Dpackaging=?(eg:jar) -Dfile=?(D:.….\alipay.*.jar)

这样就可以把mvn网络仓库没有的第三方SDK添加到本地仓库中，再在gradle依赖里面添加：

compile：”alipay:alipay:1.0”

再执行gradle.bat cleanIdea idea重新导入依赖就可以将SDK成功导入项目中。

以上注意的有：mvn本地仓库一般在用户目录下的.m2/repository下，在导入本地仓库成功后可以确认一下本地仓库是否已经有了对应生成的目录结构：repository\DgroupId\DartifactId\version，目录下是否有DgroupId-version.Dpackaging文件以及pom文件还有_rermote.repositories。如果存在则导入成功。再根据以上目录结构再在gradle里添加compile依赖。compile:”DgroupId:DartifactId:version”。以上即可成功导入。注意导入的SDK是编译好后的SDK，也就是.class文件，并非源码。
